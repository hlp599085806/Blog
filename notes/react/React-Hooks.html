    React Hooks

    # React Hooks

    # 1.useState

    [state,setState] = useState(initData)

    ​ state 提供UI视图使用的数据源

    ​ setState 改变state的函数，可以理解成推动函数组件渲染的方法

    ​ initData 定义state的初始值(分为两种情况：1.非函数、2.函数)

    1.当initData为非函数的情况 [state,setState] = useState(0)

将作为state的初始值

2.当initData为函数的情况 [state,setState] = useState( ()=>{

if(props.a===1){

return Math.random();

}else{

return Math.random()*100

}

}}

函数的返回值作为state的初始值

    ​ setState也分为两种情况: 1.非函数、2.函数

    const [state,setState] = useState(1)

1.当setState为非函数的情况

/* 点击事件 */

atClick = ()=>{

setState(2)

}

2.当setState为函数的情况

/* 点击事件 */

atClick = ()=>{

setState((state)=>state+1)

}

函数的参数是上一次的state，返回值则作为最新的state

    ​ 批量更新的机制和flushSync，在函数组件中，更新效果和类组件是一样的。

    注意点：

    ​ 1.调用改变state的函数上下文中，是获取不到最新的state的值的

    const [ number , setNumber ] = React.useState(0);
const handleClick = ()=>{
	ReactDOM.flushSync(()=>{
		setNumber(2)
	console.log(number) //0
	})
	setNumber(1)
	console.log(number) //0
	setTimeout(()=>{
		setNumber(3)
		console.log(number) //0
	})
	}

    ​ 2.使用setNumber改变number的时候，若传入相同的number(复杂类型的内存地址指向相同)，两个number的指向相同则不会发生视图更新。

    # 2.useEffect

    ​ 如何监听到state的变化?

    ​ 在类组件(classcomponent)中，可以使用setState的第二个参数callback监听到最新的state变化，或者在componentDidUpdate中看到组件更新

    ​ 在函数组件中监听useState中变化的变化，需要用到useEffect函数，useEffect函数默认会执行一次，当number发生变化的时候，useEffect就会执行。

    const [number,setNumber] = useState(1)

React.Effect(()=>{

	console.log('number是 '+number);

},[number])

    ​ 在useEffect中异步获取数据时，不应该将第一个effect参数设置为async，一旦设置为async，返回值将会是一个Promise，正确的使用方法应该是，在第一个参数的内部，写一个立即执行函数，这个立即执行函数的内部来完成async和await来获取AJAX的数据。

    ​ useEffect与componentDidMount、componentDidUpdate不同，在浏览器完成布局与绘制之后。传给useEffect的函数会延迟调用，适用于订阅或事件处理等，所以不应在函数中执行阻塞浏览器更新屏幕的操作。

    # 3.useContext

    ​ useContext是专门为函数组件提供的hook，来获取提供的contextValue。

    const value = useContext(MyContext);

    ​ 新版本创建Context的方法为 React.createContext(null);

    ​ useContext接受一个context对象(Reacy.createContext的返回值)，并返回该context的当前值。当前的context值由上层组件中距离最近的<MyContext.Provider>的valueprop决定的。

    ​ 以上是官网文档给出的结论，如果渲染组件的开销过大，可以将context的值取出来，子组件使用memo或useMemo等，将计算交给子组件，这样自己渲染的时候，如果context没有变化，依旧返回的是同样的React Element元素，这样子组件就不会进行更新渲染。

    # 4.useReducer

    const [state, dispatch] = useReducer(reducer, initialArg, init);

    ​ 了解Redux的话，这里应该就理解的很快。它是useState的代替方案，它接收一个形如(state, action) => newState的reducer，并返回当前的state以及与其配套的dispatch方法。

    ​ 在一些state逻辑比较复杂的情况下，我认为使用useReducer会比较合理。

    # 初始化state

    ​ 1.直接将初始值作为useReducer第二个参数传入

    const [state, dispatch] = useReducer(
    reducer,
    {count: initialCount}
 );

    ​ 2.如果初始值有使用到props

    function init(initialCount) {
  return {count: initialCount};
}

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return {count: state.count + 1};
    case 'decrement':
      return {count: state.count - 1};
    case 'reset':
      return init(action.payload);
    default:
      throw new Error();
  }
}

function Counter({initialCount}) {
  const [state, dispatch] = useReducer(reducer, initialCount, init);
  return (
    <>
      Count: {state.count}
      <button
        onClick={() => dispatch({type: 'reset', payload: initialCount})}>
        Reset
      </button>
      <button onClick={() => dispatch({type: 'decrement'})}>-</button>
      <button onClick={() => dispatch({type: 'increment'})}>+</button>
    </>
  );
}

    # 跳过dispatch修改state

    ​ useReducer和useState一样，在dispatch的时候，都采取的是浅比较(底层是Object.is)来对比两个state,如果需要更新，可以搭配拓展运算符来使用。

    # 5.useCallback

    const memoizedCallback = useCallback(
  () => {
    doSomething(a, b);
  },
  [a, b],
);

    ​ 返回一个 [memoized]回调函数，将callback和依赖传入useCallback，将返回一个缓存过的回调函数，只有当依赖项发生改变的时候，这个返回的回调函数才会发生更新。

    # 6.useMemo

    const memoizedValue = useMemo(
    () => computeExpensiveValue(a, b)
    , 
[a, b]
);

    ​ 返回一个memoized值。第一眼看下来，跟useCallback完全一样，但是不一样的是，useCallback是返回缓存的函数，useMemo保存的是callback中return的内容，但是和useCallback一样，当依赖值发生变化的时候，才会重新计算。

    # 注意！！！

    ​ 传入useMemo的函数，会在渲染期间执行，禁止在这个函数中执行与渲染无关的副操作。

    # 7.useRef

    const refContainer = useRef(initialValue);

    ​ useRef返回一个可变的ref对象，其.current属性被初始化为传入的参数，返回的ref对象在没有进行改变操作的话，整个生命周期内保持不变。

    /* 一个最简单的例子 */
function TextInputWithFocusButton() {
  const inputEl = useRef(null);
  const onButtonClick = () => {
    // `current` 指向已挂载到 DOM 上的文本输入元素
    inputEl.current.focus();
  };
  return (
    <>
      <input ref={inputEl} type="text" />
      <button onClick={onButtonClick}>Focus the input</button>
    </>
  );
}
