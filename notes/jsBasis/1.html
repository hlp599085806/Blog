    # JavaScript深入之从原型到原型链

    # 构造函数创建对象

    先使用构造函数创建一个对象：

    function Person(){

....

}
let person = new Person();
person.name ='hlp';
console.log(person.name); // hlp

    在这个例子中，Person就是一个构造函数，和普通的函数不同，首字母大写是构造函数的规范，我们再使用了new创建了一个实例对象person。

    # prototype

    每个函数都有一个prototype属性，比如：

    function Person(){

...
}
//prototype是函数才会有的属性!!!!
Person.prototype.name='hlp';
let p1 = new Person();
let p2 = new Person();
p1.name === p2.name;// true

    那么函数的prototype属性代表什么，指向什么。

    其实函数的prototype属性指向了一个对象，这个对象正是调用该构造函数而创建的实例的原型，上述代码中p1、p2的原型。

    那什么是原型呢？我认为可以这么理解：每一个JavaScript对象(null除外)在创建的时候就会与之关联一个对象，这个对象就是我们所说的原型，每一个对象都会从原型"继承"属性。

    现在了解了函数与实例原型的关系，那么实例与实例原型之间的关系呢(p1和Person.prototype)?

    # _proto_

    这是每一个JavaScript对象(null除外)都具有的一个属性，叫__proto__，这个属性会指向该对象的原型。

    function Person(){
...
}
let p1 = new Person();
p1.__proto_\_ === Person.prototype;// true

    既然函数和实例对象都可以指向原型，那么原型是否也可以指向函数或实例对象呢？

    # constructor

    原型指向实例没有，毕竟一个构造函数可以创建多个实例。但是原型指向构造函数倒是有，原型constructor属性，每个原型都有一个constructor属性指向构造函数。

    function Person(){
...
}
Person === Person.prototype.constructor // true

    目前的三个属性我们可以得到很多关系:

    function Person() {
...
}
let person = new Person();
console.log(person.__proto_\_ == Person.prototype) // true
实例对象的__proto__和函数的prototype都指向原型
console.log(Person.prototype.constructor == Person) // true
原型的constructor指向构造函数
// 顺便学习一个ES5的方法,可以获得对象的原型
console.log(Object.getPrototypeOf(person) === Person.prototype) // true

    # 实例与原型

    当读取实例属性时，如果找不到，则会查找与实例对象相关联的原型中去查找属性，如果还查找不到，则会去原型的原型，一直找到最顶层为止。

    function Person() {
...
}
Person.prototype.name = 'Kevin';
let person = new Person();
person.name = 'Daisy';
console.log(person.name) // Daisy
delete person.name;
console.log(person.name) // Kevin

    当实例对象自己存在属性的时候，自然是拿自己的属性，当实例对象的属性不存在时，就会去person的原型，也就是person.__proto__或Person.prototype中查找属性。如果原型也没有呢？去原型的原型查找吗？有终点吗?

    # 原型的原型

    前面讲到，原型也是一个对象，既然是对象，我们就可以使用最原始的方式去创建。

    let a = new Object();

let b = {};

    其实原型对象就是通过object构造函数生成的。

    # 原型链

    那 Object.prototype 的原型呢？

    null

console.log(Object.prototype.__proto_\_ === null) // true

    null代表着没有对象，也就是Object.prototype 没有原型。

    图中由相互关联的原型组成的链状结构就是原型链，也就是蓝色的这条线

    # 补充

    # constructor

    function Person() {
...
}

let person = new Person();

console.log(person.constructor === Person); // true

    当获取 person.constructor 时，其实 person 中并没有 constructor属性,当不能读取到constructor 属性时，会从 person 的原型也就是 Person.prototype中读取，正好原型中有该属性。

    person.constructor === Person.prototype.constructor // true

    # _proto_

    其次是 _proto_，绝大部分浏览器都支持这个非标准的方法访问原型，然而它并不存在于Person.prototype 中，实际上，它是来自于 Object.prototype，与其说是一个属性，不如说是一个 getter/setter，当使用 person._proto_时，可以理解成返回了 Object.getPrototypeOf(person)。
