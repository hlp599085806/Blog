    # JavaScript深入之执行上下文栈

    JS代码的执行顺序，第一直觉就是按照代码顺序执行，例如如下代码：

    var a = function(){
console.log(1);
}
a();//1
var a = function(){
console.log(2);
}
a() //2


    function a(){
console.log(1);
}
a() // 2
function a(){
console.log(2);
}
a() // 2

    两个代码打印的结果却不相同，这是因为JS代码并非是一行一行执行的，而是一段一段执行的。当执行到一段代码的时候，会进入一个执行上下文的建立阶段，例如第一段的变量提升，第二段代码的函数提升。

    # 可执行代码

    1.全局代码 2.函数代码 3.eval代码

    当执行到一个函数的时候，就会进入到执行上下文状态中。

    # 执行上下文栈

    JS引擎是怎么帮我们管理这么函数的上下文的呢？

    JS引擎创建了执行上下文栈来管理执行上下文。由于是一个栈，我们用数组来模拟这个执行上下文栈。

    ECStack = []; // 执行上下文栈(数组)

    当开始运行JS代码的时候，首先先进入的就是全局环境，所以就把全局上下文栈压入到执行上下文栈中，并且只有当整个应用程序结束后，才会清空这个执行上下文栈。
