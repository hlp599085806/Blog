    TypeScript接口(interface）

    # TypeScript接口(interface）

    TypeScript的核心原则之一是对值所具有的结构进行类型检查,它有时被称做“鸭式辨型法”或“结构性子类型化”。

    在TypeScript里，接口的作用就是为这些类型命名和为你的代码或第三方代码定义契约

    ​ 例如一个函数接受一个对象，然后函数返回这个对象的某个属性：

    const getUserName = (info)=>info.name;

    ​ 这样的写法在js中是没有任何问题的，但是在ts的开发环境中，是非常不推荐的。需要使用一种类型去描述这个参数。

    ​ 我们就可以使用interface

    interface userInfoData {
    name: string;
    age: number;
}

    ​ 这个接口描述了参数的结构，接口不会去检查属性的顺序，只需要相应的属性存在并且类型兼容即可。

    ​ 这时候又出现一个问题了，当后端接口返回数据时，根据不同的id去查询内容的时候，可能会返回不同的属性。

    # 可选属性

    ​ 比如这个userInfoData参数，有的用户存在手机号，有的不存在；

    interface userInfoData {
    name: string;
    age: number;
    phoneNumber?: number; 
}

    ​ 这时候phoneNumber的类型可能是number类型也可能是undefined类型。

    # 只读属性

    ​ 我们的用户性别是不允许修改的，可以使用readonly，这样就无法更改属性了

    # 函数类型

    ​ 假设这个userInfoData中有一个函数，interface怎么定义呢？

    interface userInfoData {
    name: string
    age?: number
    readonly isMale: boolean
    say: (words: string) => string
}

    ​ 又或者我另外用接口直接描述函数类型

    interface Say {
    (words: string) : string
}

    ​ 然后在userInfoData中使用

    interface userInfoData {
    name: string
    age?: number
    readonly isMale: boolean
    say: Say
}

    # 属性检查

    ​ 当对象字面量当被赋值给变量或作为参数传递的时候，会被特殊对待而且经过“额外属性检查”。 如果一个对象字面量存在任何“目标类型”不包含的属性时，你会得到一个错误。

    interface Config {
  width?: number;
}

function  CalculateAreas(config: Config): { area: number} {
  let square = 100;
  if (config.width) {
      square = config.width * config.width;
  }
  return {area: square};
}
// error: 'widdth' not expected in type 'Config'
let mySquare = CalculateAreas({ widdth: 5 });

    ​ 解决方案：

    ​ 1.类型断言

    ​ 2.字符串索引

    ​ 3.字面量赋值给另一个变量

    # 继承接口

    ​ 假设又有一个新的接口，需要一个parentUserInfoData，这个接口除了userInfoData中的属性，还额外新增了两个属性,这个时候就可以使用extend去完成继承。

    interface parentUserInfoData extends User {
    address: string;
    getChildren: () => void;
}
